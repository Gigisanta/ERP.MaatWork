# Claude Project Configuration Example

This file shows how to integrate the universal engineering rules into a Claude Project configuration.

## Integration for Claude Desktop App

Create or update `.claude_project.toml`:

```toml
[config]
name = "Cactus CRM"
url = "https://github.com/your-org/your-repo"

[ai_model]
model = "claude-3-5-sonnet-20241022"
temperature = 0.1

[context]
files = [
  "AI_AUTO_DEV_RULES.md",
  "SYSTEM_NOTES.md",
  "ROBUSTIFY_RULES.md",
  "MEMORY_ENGINE.md"
]

[prompt]
system_prompt = """
You are an AI assistant following universal engineering standards.

CRITICAL: Before any code modification, read and apply the 30 rules from AI_AUTO_DEV_RULES.md

Decision Flow:
1. Is this change necessary? NO → Stop
2. Does it follow rules 1-30? NO → Refactor
3. Is it safe? NO → Add tests first
4. Will it break existing code? YES → Version it
5. Is it documented? NO → Add inline comments

Priority: When in doubt, choose the MORE STABLE solution, not the more creative one.

Key Rules to Always Follow:
- Rule 12: Fail fast with clear messages
- Rule 13: Tests before refactoring
- Rule 15: Small, atomic changes
- Rule 22: Document architecture decisions
- Rule 27: Maintain risk list before large refactors

Format for changes:
// AI_DECISION: [Motivation]
// Justification: [Why this is better]
// Impact: [What areas of code are affected]
"""
```

## Integration for Claude API

When using Claude API, include in your system prompt:

```
Reference AI_AUTO_DEV_RULES.md for all engineering decisions.

Core directive: When in doubt between two solutions, choose the more stable one, not the more creative one.

Follow the decision flow from AI_AUTO_DEV_RULES.md before any code change.
```

## Files to Include

Make sure these files are in your project:

1. `AI_AUTO_DEV_RULES.md` - Main rules file (required)
2. `README_SUPERPROMPT.md` - Complete guide
3. `SYSTEM_NOTES.md` - Your project context (optional but recommended)
4. `MEMORY_ENGINE.md` - Historical decisions (optional but recommended)

---

## Quick Reference Commands

```bash
# Verify files exist
ls -la AI_AUTO_DEV_RULES.md README_SUPERPROMPT.md

# Check integration
grep "AI_AUTO_DEV_RULES" .claude_project.toml .cursorrules
```

## Testing the Integration

Test with a prompt like:
"Add a new user registration endpoint following the universal engineering rules"

The AI should:
1. Check AI_AUTO_DEV_RULES.md
2. Propose a stable solution
3. Add tests
4. Document decisions inline
5. Follow naming conventions
