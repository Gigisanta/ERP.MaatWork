# Replit Auto-dev Integration Example

This file shows how to integrate the universal engineering rules into Replit's Auto-dev mode.

## Configuration File

Create `.replit` file in your project root:

```toml
modules = []

[deploy]
run = ["bash", "-c", "echo 'Deploy configuration'"]

[env]
VIRTUAL_ENV = "/home/runner/venv"

[languages]

[languages.javascript]
languageServer = {
  start = "node /opt/replit/javascript-language-server/server.js --stdio"
}

[languages.typescript]
languageServer = {
  start = "typescript-language-server --stdio"
}

[packager]
language = "npm"
ignoredPackages = [
  "unit_tests"
]
ignoredPaths = [
  "node_modules"
]

[packager.features]
enabledForHosting = false
packageSearch = true
guessImports = true
enabledForWebTesting = true

[languages.javascript.languageServer]
start = "node /opt/replit/javascript-language-server/server.js --stdio"

[AI]
systemPrompt = """
You are an AI assistant following universal engineering standards for this Replit project.

CRITICAL INSTRUCTIONS:

1. Read AI_AUTO_DEV_RULES.md before any code modification
2. Follow the 30 universal rules for all changes
3. Decision Flow:
   - Is this change necessary? NO → Stop
   - Does it follow rules 1-30? NO → Refactor
   - Is it safe? NO → Add tests first
   - Will it break existing code? YES → Version it
   - Is it documented? NO → Add comments inline

PRIORITY: When in doubt between two solutions, choose the MORE STABLE one, not the more creative.

Key Rules:
- Rule 12: Fail fast with clear messages
- Rule 13: Tests before refactoring
- Rule 15: Small, atomic changes
- Rule 22: Document architecture decisions
- Rule 27: Maintain risk list before large refactors

Format for code changes:
// AI_DECISION: [Reason for change]
// Justification: [Why this approach]
// Impact: [What code areas affected]

Always use structured logging and proper error handling.
"""
```

## Auto-dev Configuration

Create `replit-ai-rules.md` in your project root:

```markdown
# Replit Auto-dev Rules

This project uses universal engineering standards from AI_AUTO_DEV_RULES.md.

## Quick Reference

### Before Making Changes
1. Check AI_AUTO_DEV_RULES.md
2. Ensure tests pass
3. Run linter
4. Document decisions

### Code Format
```typescript
// AI_DECISION: Reason
// Justification: Why
// Impact: Scope
```

### Priority
Stability > Creativity
Clear > Clever
Safe > Fast

### Key Files
- AI_AUTO_DEV_RULES.md: Main rules
- SYSTEM_NOTES.md: Project context
- MEMORY_ENGINE.md: Historical decisions
```

## Integration Steps

1. **Add Rules Files to Replit**:
   ```bash
   # Upload these files to your Replit project
   wget https://github.com/your-repo/AI_AUTO_DEV_RULES.md
   # Or copy them manually
   ```

2. **Update `.replit`**:
   - Add the AI system prompt above
   - Reference the rules files

3. **Test Integration**:
   ```bash
   # Ask Replit AI to refactor some code
   # It should follow the rules automatically
   ```

## Files to Include in Replit

Required:
- `AI_AUTO_DEV_RULES.md` ✅
- `.replit` ✅

Recommended:
- `README_SUPERPROMPT.md` - Full guide
- `SYSTEM_NOTES.md` - Project context
- `MEMORY_ENGINE.md` - History

## Auto-dev Prompt Example

When using Auto-dev in Replit, the AI will automatically:

1. Check `AI_AUTO_DEV_RULES.md` before changes
2. Follow the decision flow
3. Choose stable solutions
4. Add proper documentation
5. Include tests where needed

Example prompt:
```
"Add user authentication with JWT tokens following the universal rules"
```

The AI will:
- ✅ Check rules 1-30 first
- ✅ Propose a stable JWT implementation
- ✅ Add tests
- ✅ Document inline
- ✅ Follow naming conventions

```
